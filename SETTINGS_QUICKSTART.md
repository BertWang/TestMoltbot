## 🎯 常见任务

### 任务 1: 配置 AI 模型

1. 打开 `/settings`
2. 点击 **AI 设定** 标签页
3. 选择模型按钮 (推荐: Gemini 2.0 Flash)
4. 点击 **保存配置**

✅ **完成**: AI 模型已配置

---

### 任务 2: 查看系统状态

1. 打开 `/settings`
2. 默认显示 **概览** 标签页
3. 查看卡片信息:
   - AI 提供商 (显示当前配置)
   - 已启用的整合数 (显示进度)

✅ **完成**: 了解系统状态

---

### 任务 3: 获取 AI 建议

1. 打开 `/settings`
2. 点击 **快速操作** 中的 **AI 助手**
3. 等待 openclaw.ai 分析
4. 查看推荐项、问题项、优化建议
5. 点击 **应用建议** 或 **关闭**

✅ **完成**: 获得个性化配置建议

---

### 任务 4: 添加整合服务

1. 打开 `/settings`
2. 点击 **整合** 标签页
3. 滚动到底部
4. 点击 **新增整合服务**
5. 选择服务类型
6. 按照向导完成配置

✅ **完成**: 整合服务已添加

---

### 任务 5: 启用/禁用整合

1. 打开 `/settings`
2. 点击 **整合** 标签页
3. 找到要修改的服务
4. 点击 **启用** 或 **禁用** 按钮

✅ **完成**: 整合状态已更改

---

```bash
# .env.local
GEMINI_API_KEY=your_key_here
AI_PROVIDER=gemini
GEMINI_MODEL=gemini-2.0-flash-exp
```

### MCP 服务器配置

```json
{
  "provider": "mcp-server",
  "enabled": true,
  "config": {
    "command": "npx",
    "args": ["-y", "@notionhq/notion-mcp-server"],
    "env": {
      "NOTION_API_KEY": "__ENV__NOTION_API_KEY__"
    }
  }
}
```

## 🆘 故障排除

### 问题 1: openclaw.ai 分析不显示

**原因**: API 可能不可用  
**解决方案**:
1. 检查网络连接
2. 刷新页面
3. 使用本地备用分析 (自动启用)

### 问题 2: AI 模型保存失败

**原因**: 网络错误或验证失败  
**解决方案**:
1. 检查模型名称格式
2. 重新尝试保存
3. 查看浏览器控制台错误

### 问题 3: 整合服务无法启用

**原因**: 配置不完整或 API 问题  
**解决方案**:
1. 检查配置是否有效
2. 查看错误提示信息
3. 重新启动整合

### 问题 4: 页面响应缓慢

**原因**: 分析计算过程中  
**解决方案**:
1. 等待分析完成 (通常 < 2 秒)
2. 减少配置项
3. 清除浏览器缓存
